Act as an expert evaluator of code review quality.

TASK

You will be given:
1) A source code snippet
2) A target code smell identified by a human reviewer (ground truth)
3) A code review generated by an LLM

Your task is to classify the LLM response into ONE of the following categories:
- Instrumental (I)
- Helpful (H)
- Misleading (M)
- Uncertain (U)

You must base your decision on how well the LLM response matches the target code smell identified in the code.

Your evaluation must consider:
- defect type (code smell)
- defect location (when applicable)
- defect description

DEFINITIONS

Instrumental (I):
The type of the defect identified in the response is exactly the same as the target defect, the location is strictly within the relevant code region, and the description provides effective and accurate information.

Helpful (H):
The type or location differ to some extent, or the description is partially inaccurate, but still provides useful evidence toward the target defect.

Misleading (M):
The response does not provide any helpful information, fails to detect the target defect, or claims no issues.

Uncertain (U):
The response does not mention the target defect and instead points out other possible issues that cannot be confirmed.

IMPORTANT RULES

- You must choose ONLY ONE category.
- Be strict when assigning Instrumental.
- If the response partially matches but is not exact, prefer Helpful.
- If the response misses the target defect entirely, prefer Misleading.
- If the response discusses unrelated issues, prefer Uncertain.
- Do NOT assume context beyond what is given.
- The target defect is the code smell identified by the human reviewer.

OUTPUT FORMAT

Category: [I | H | M | U]
Justification: Brief explanation based on type, location, and description.
